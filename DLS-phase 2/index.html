Logic Explanation

The lootArray serves as the single source of truth for the application. Every loot item is stored as an object containing a name, value, and quantity. The quantity property was added to support multiple copies of the same loot item.

The updateUI() function is responsible for calculating totals, calculating loot per party member, rendering the loot list, enabling or disabling buttons, and controlling visibility of results.

The application follows a state-driven design. Whenever the loot array or party size changes, updateUI() runs and refreshes the interface automatically.

Conditional logic prevents invalid state by validating party size, loot names, loot values, and quantity before modifying the loot array.
